# VideoFlow PRD v1.0 (영상 제작)

문서번호: VF-PRD-2026-001
작성일: 2026-06-03
상태: 골격 초안 (MemoryFlow MVP 완성 후 상세화)
연관 문서: MemoryFlow PRD v3.0 (MF-PRD-2026-003)

---

## 0. 이 문서의 위치

VideoFlow는 MemoryFlow에서 내보낸 제작용 패키지를 받아 짧은 스토리 영상을 렌더링하는 영상 제작 측 시스템이다.

이 문서는 골격만 잡아둔 상태다. 실제 상세 설계는 **MemoryFlow 앱을 완성한 뒤** 진행한다. 지금은 두 시스템의 경계와 입출력 계약만 명확히 해둔다.

### 책임 경계

```text
MemoryFlow가 책임지는 것 (MemoryFlow PRD §11)
- 승인된 스토리북을 제작용 패키지로 내보내기
- 외부에서 만들어진 완성 MP4를 다시 받아 보관·공유

VideoFlow가 책임지는 것 (이 문서)
- 제작용 패키지 가져오기
- 패키지를 바탕으로 영상 렌더링
- 완성 MP4를 MemoryFlow에 전달 (수동 업로드)
```

즉 MemoryFlow는 "패키지를 만들어 건네고 완성본을 받는다"까지, VideoFlow는 "패키지를 받아 영상으로 만든다"를 맡는다.

---

## 1. 전체 흐름

```text
MemoryFlow에서 스토리북 승인 (장면·순서·내레이션·사진 선별 확정)
-> MemoryFlow가 제작용 패키지 내보내기
-> [VideoFlow] 제작 장비에서 패키지 가져오기
-> [VideoFlow] 패키지 기반으로 영상 렌더링
-> [VideoFlow] 완성 MP4 출력
-> 관리자가 완성 MP4를 MemoryFlow에 수동 업로드
```

---

## 2. 입력 — 제작용 패키지 (MemoryFlow가 생성)

VideoFlow의 입력은 MemoryFlow가 내보내는 패키지다. 이 구조는 MemoryFlow PRD §11과 일치해야 한다(입출력 계약).

```text
project.json          - 프로젝트 메타데이터
scene-timeline.json   - 장면별 길이, 사진 순서, 내레이션 텍스트
이미지/영상 파일 묶음   - 실제 미디어 에셋
커버 이미지
(BGM은 Phase 2)
```

### scene-timeline.json 의 핵심

MemoryFlow의 영상 길이 모델(MF-PRD §8)과 1:1로 대응한다. 장면 하나 = 타임라인의 한 구간.

```text
장면(scene)마다:
- 장면 길이(초)        = (사진 수 × 사진 노출시간) + 영상 클립 길이
- 사진/영상 순서
- 내레이션 텍스트       = 그 장면 길이에 맞게 다듬어진 최종 내레이션
```

> 핵심: 길이·순서·내레이션이 MemoryFlow 승인 시점에 이미 확정되어 넘어온다. VideoFlow는 이를 "해석해서 그릴" 뿐, 내용을 다시 편집하지 않는다.

---

## 3. 렌더링 (상세는 추후)

제작 장비에서 패키지를 가져와 `scene-timeline.json`의 타임라인대로 장면을 이어붙여 영상을 만든다.

```text
장면 단위로 렌더링:
- 그 장면의 사진/영상이 정해진 순서로 흐름
- 사진은 장면 길이 안에서 노출시간만큼 표시
- 내레이션이 자막/음성으로 깔림
- 장면들을 순서대로 이어 전체 영상 완성
```

구체적인 렌더링 도구·템플릿·트랜지션·자막 스타일·음성(TTS) 사용 여부 등은 **MemoryFlow 완성 후 이 문서에서 상세화한다.**

---

## 4. 출력

```text
완성 MP4
- 관리자가 MemoryFlow에 수동 업로드 (MF-PRD §11)
- 이후 MemoryFlow가 공유 링크로 배포
```

## 6. 메모

MemoryFlow와 VideoFlow의 유일한 결합점은 **제작용 패키지(입력)와 완성 MP4(출력)** 두 가지다. 이 두 계약만 안정적으로 유지되면 두 시스템은 독립적으로 발전할 수 있다. VideoFlow 상세 설계는 MemoryFlow가 패키지를 실제로 내보내기 시작한 뒤, 실물 패키지를 보면서 진행하는 것이 가장 정확하다.
