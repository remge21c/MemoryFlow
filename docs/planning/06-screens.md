# MemoryFlow 화면 인벤토리 (Screen Inventory)

문서번호: MF-SCREEN-2026-001
작성일: 2026-06-03
근거: memoryflow-prd.md v3.0 (검증 완료) / 3장 역할·권한에서 연역
다음 단계: /screen-spec → specs/screens/*.yaml

> **전역 제약: 모바일 우선(Mobile-First).** 모든 화면은 모바일 화면을 1순위로
> 설계하고, 데스크톱은 확장으로 본다. 업로더는 현장에서 폰으로 올리고,
> 관리자도 폰으로 운영 가능해야 한다. 디자인 시스템(PRD 15장)의 단일 출처를 따른다.

> 연역 원칙: 모든 화면은 3장의 역할(관리자/업로더/공유)에서 도출된다.
> PRD에서 정당화되지 않는 화면은 넣지 않는다(Feature Creep 차단).

---

## 0. 사이트맵 (역할별)

```text
[공통]
  로그인 (전화번호 + 비밀번호)
  초대 진입 → 가입 (/join/:token → 이름·전화번호·비밀번호)

[업로더]
  내 프로젝트 목록
  └ 프로젝트 일정 목록 (Day → 장면)
     └ 기여 업로드/수정 (사진·영상 + 스토리)
  결과물 열람 → (공유 링크로 이동)

[관리자]
  프로젝트 목록 (대시보드)
  └ 프로젝트 생성
  └ 프로젝트 상세 (Day 목록)
     ├ 세부일정(장면) 설계
     ├ 초대 링크 관리
     ├ 멤버 관리
     ├ 스토리북 편집 ★핵심
     │   └ 승인 / 편집 잠금
     ├ 공유 링크 관리
     ├ 영상 패키지 내보내기
     └ 최종 영상 업로드/관리

[공유 · 비로그인]
  /share/:token  스토리북 열람 + 최종 영상 시청
```

---

## 1. 공통 화면

### S-01 로그인
- 입력: 전화번호 + 비밀번호 (PRD 4장)
- 진입: 직접 방문, 또는 초대 링크에서 기존 계정일 때
- 출력: 역할에 따라 관리자 대시보드 / 업로더 프로젝트 목록

### S-02 초대 진입 → 가입 (`/join/:token`)
- 토큰 검증(만료·무효 확인) → 유효 시 가입 폼
- 계정 없음: 이름 + 전화번호 + 비밀번호 입력 → 자동으로 해당 프로젝트 업로더 합류
- 계정 있음: 로그인만 하면 해당 프로젝트에 합류
- 만료/무효 토큰: 안내 화면

---

## 2. 업로더 화면

### S-10 내 프로젝트 목록
- 본인이 합류한 프로젝트 카드 목록 (계정 1개로 여러 프로젝트, PRD 4장)
- 각 카드: 프로젝트명·단체·기간·상태

### S-11 프로젝트 일정 목록 (Day → 장면)
- 관리자가 설계한 Day별 세부일정(장면) 목록을 본다
- 업로더는 고르기만 하면 됨 (PRD 5장 "부담 적음")
- 장면별 본인 기여 유무 표시

### S-12 기여 업로드/수정
- 장면 1개 선택 → 사진·영상 + 스토리(텍스트) 업로드
- 같은 장면에 이미 올라온 타인의 사진/영상 미리 보임 (중복 보여주기, 차단 안 함, PRD 6장)
- 본인 기여만 수정/삭제 (승인 전, PRD 3장)
- 편집 잠금/승인 후: 읽기 전용

### S-13 결과물 열람
- 별도 화면 없음 — 승인된 스토리북·최종 영상은 **공유 링크(S-30)**로 본다 (PRD 3장)

---

## 3. 관리자 화면

### S-20 프로젝트 목록 (대시보드)
- 관리하는 프로젝트 목록 + 상태(active/completed/archived)
- 신규 프로젝트 생성 진입

### S-21 프로젝트 생성
- 입력: 프로젝트명·소속/단체명·설명·대표이미지(직접 업로드)·시작일·종료일·기본 사진 노출시간 (PRD 5장)
- 저장 시 시작~종료일로 Day 자동 생성

### S-22 프로젝트 상세 (Day 목록)
- Day 목록 + 각 Day 진입
- 프로젝트 운영 메뉴 허브 (일정설계·초대·멤버·스토리북·공유·영상)

### S-23 세부일정(장면) 설계
- Day별 세부일정 CRUD
- 항목: 시간·제목·장소·분류·정렬순서·사진 노출시간(장면별 덮어쓰기, nullable)
- "장면 = 영상 대본의 한 컷"임을 인지시키는 UI (PRD 5장)

### S-24 초대 링크 관리
- 프로젝트별 초대 링크 발급(`/join/:token`)·만료시간 설정·무효화 (PRD 4장)

### S-25 멤버 관리
- 합류한 멤버 목록 (자동 표시)
- 사후 내보내기/비활성화(active/inactive), 임시 비밀번호 설정 (PRD 4장)

### S-26 스토리북 편집 ★핵심 화면
- **편집 단위 = 세부일정(장면) 하나** [확정 T-2]. 한 화면에서 장면 1개만 집중 편집하고,
  이전/다음 장면으로 이동(모바일 우선에 적합). 전체 스토리북 조망은 별도 목록/미리보기로.
- PRD 7장 "편집 화면 개념"을 그대로 구현
- 장면 1개를 열면 업로더 구분 보이는 채로 기여 펼침
- 기능 3종:
  1. 사진/영상 선별 — 포함/제외 토글 (제외해도 흐리게 남아 되돌리기)
  2. 순서 조정 — 드래그
  3. 스토리 합치기 — [대본 초안 생성](AI) → 내레이션 편집 → [길이에 맞게 요약](AI)
- 예상 장면 길이 표시 (사진수×노출시간 + 클립길이, PRD 8장)
- 적정 내레이션 글자수 가이드(제안, 강제 아님)
- AI 호출: 로딩·30초 타임아웃·에러 토스트·[다시 시도]·[적용] 분리 (PRD 9장)

### S-27 승인 / 편집 잠금
- 편집 잠금 토글(승인 전 업로더 입력 차단, PRD 7장)
- 스토리북 승인 → 전체 잠금 / 관리자만 잠금 해제

### S-28 공유 링크 관리
- 공유 링크 발급·만료옵션(30/60/120/180/360일, 기본 30)·무효화 (PRD 12장)

### S-29 영상 패키지 내보내기 / 최종 영상 관리
- 승인된 스토리북 → 영상 제작용 패키지 내보내기(project.json·scene-timeline.json·미디어, PRD 11장)
- 외부 제작 완성 MP4 업로드·상태 관리(uploaded/published/hidden)

---

## 4. 공유 화면 (비로그인)

### S-30 공유 스토리북 열람 (`/share/:token`)
- 토큰 해시 검증 → 승인된 공개 스토리북만 표시 (PRD 12장)
- Day → 장면 → 사진/영상 흐름 + 합쳐진 내레이션
- 공개된 최종 영상 시청 (HTTP Range 인라인 재생, PRD 13장)
- 허용 안 함: 로그인·업로드·댓글·편집·비공개 조회 (PRD 3장)

## 5. 열린 결정 (Taste Decision) — 확정됨 [2026-06-03]

> 코기토에서 자동 연역되지 않던 화면 결정들. 사용자 확정 완료.

| # | 결정 | 확정 |
|---|------|------|
| T-1 | 업로더 일정 목록(S-11) 기본 뷰 | **Day별 리스트** |
| T-2 | 스토리북 편집(S-26) 장면 이동 | **세부일정(장면) 단위 편집** + 이전/다음 이동 |
| T-3 | 관리자 프로젝트 상세(S-22) 메뉴 | **탭, 모바일 우선** |
| T-4 | 기여 업로드(S-12) 사진+스토리 | **한 폼에 동시 입력** |

> **T-3 후속 메모(디자인 시스템 단계):** 모바일에서 6개 운영 메뉴를 상단 탭으로
> 다 노출하면 비좁다. 가로 스크롤 탭 또는 하단 시트/더보기로 접는 패턴을
> 디자인 시스템(PRD 15장)에서 확정한다. "탭" 의도(고정·명확)는 유지.
